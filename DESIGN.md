> 方向已定：**「北漂叙事」— 暗色 + 辣椒红 + 滚动叙事**
> 身份：岳阳人 × 北大学社会学 × 个人经验站

---

## 一、设计系统

### 配色

```
最深背景    #0A0A0A    全屏 hero、页脚
主背景      #111111    内容区
次级背景    #1A1A1A    卡片、分隔区
悬浮背景    #222222    hover 状态

主文字      #F5F0EB    暖白（不是冷白，更柔和）
次要文字    #9E9E9E    灰色说明文字
暗淡文字    #666666    日期、标注

辣椒红      #FF3B3B    唯一强调色——链接、按钮、重点标记
辣椒红hover #FF5555    悬停状态
金色点缀    #FFB800    极少使用，仅用于特别重要的高亮

边框线      #2A2A2A    微妙的分割
```

### 字体

```
标题：Noto Serif SC（思源宋体）— 大字用衬线体，有分量感
正文：Noto Sans SC（思源黑体）— 阅读舒适
数字/代码：JetBrains Mono — 偶尔用于数据标注

字号梯度（桌面端）：
  .hero-title    clamp(3rem, 8vw, 8rem)    超大标题
  .section-title clamp(2rem, 5vw, 4rem)    章节标题
  .body-large    clamp(1rem, 2vw, 1.25rem)  大段文字
  .body          1rem                        正文
  .caption       0.875rem                    说明文字
```

### 间距与节奏

```
章节之间：大留白（120px-200px），让每个章节"呼吸"
章节内部：内容紧凑，有节奏感
滚动触发点：每个章节的核心内容在视口 30% 处触发动画
```

## 二、首页设计（长滚动叙事）

首页就是网站的主体。用户从顶部滚到底部，经历一个湖南人来北京的完整叙事。

### Section 0 — 导航栏

```
固定在顶部，背景从透明 → 滚动后渐变为 #0A0A0A（半透明毛玻璃）
左侧：🌶️ 图标 + "生存指南" 四个字
右侧：6 个板块快捷链接（小字）+ "关于"
移动端：左侧图标，右侧汉堡菜单

极简、不抢戏。悬停时链接变辣椒红色。
```

### Section 1 — Hero（首屏）

```
占满整个屏幕（100vh），纯黑背景

居中排列：
  一行小字（辣椒红）："一个岳阳人的"
  超大标题（白色衬线体）："北京生存指南"
  一行副标题（灰色）："从洞庭湖到未名湖，从岳阳楼到博雅塔"
  
向下滚动指示器：底部中间，一个微妙的向下箭头，微微上下浮动

背景：极简粒子/噪点纹理，或是一片微弱的红色光晕在底部

整个 Hero 区没有按钮、没有导航、没有多余信息。
就三行字 + 一个箭头。让访客自己决定往下滚。
```

### Section 2 — "出发"

```
触发：滚动到这一节时，文字从下方浮现

大标题："202X 年夏天，我坐上了去北京的火车"
  （具体年份你来填）

左边一大段文字（个人叙事）：
  - 岳阳是什么样的，你为什么来北京
  - 来之前对北京的想象 vs 现实
  - 第一印象

右边（或背景）一张图：
  - 可以是岳阳的图 vs 北京的图
  - 或者一张火车窗外的风景

视觉处理：
  文字逐段浮现（Framer Motion fade-in + slide-up）
  图片从右边滑入
  整体暗色，图片做了暗角处理融入背景
```

### Section 3 — "第一口北京的空气"（气候）

```
大标题："北京没有岳阳的湿润"

这个章节是"冲击感"——气候是湖南人来北京最直接的震撼

内容分三块，用卡片或横向滑动展示：

卡片 1：沙尘暴
  "第一次经历沙尘暴，我以为世界末日了"
  辣椒评级：🌶️🌶️🌶️🌶️🌶️（5/5，跟湖南完全不同）

卡片 2：杨絮
  "春天下的不是雨，是棉花"
  🌶️🌶️🌶️🌶️

卡片 3：干燥
  "来北京第一周，我流了三次鼻血"
  🌶️🌶️🌶️🌶️

每张卡片：
  暗色背景（#1A1A1A）
  左上角辣椒评级
  中间大字体验描述
  下方小字实用建议

视觉处理：
  三张卡片从左到右排列（桌面）或上下堆叠（移动端）
  hover 时卡片微微上浮 + 红色边框出现
```

### Section 4 — "北京很大"（生活）

```
大标题："北京有 16 个区，我住海淀"

内容方向：
  - 北大的宿舍/周边是什么样的
  - 北京地铁初体验（和岳阳的交通对比）
  - 租房的话，各区怎么选

视觉处理（方案 A——简洁卡片）：
  一个 2×3 的 Bento Grid
  每格一个生活话题的入口（租房/交通/医保/办证/购物/通信）
  格子大小不一，重点话题占大格
  点击跳转到独立的详情页

视觉处理（方案 B——地铁图隐喻）：
  用北京地铁线路图的美学来做这一节的背景
  抽象化的线条 + 站点名
  内容浮在线上
```

### Section 5 — "舌尖上的乡愁"（美食）⭐

```
大标题："来北京之后，我才知道剁椒是可以拿来炫耀的"

这是整站的情感核心——对湖南人来说，吃是最大的乡愁

内容结构：
  
  上半部分：北京湘菜馆吐槽/推荐
    - 一张北京地图（高德地图集成）
    - 标注你知道的好吃的湘菜馆
    - 点击弹出详情：店名、推荐菜、为什么好
  
  下半部分：北京食物的湖南人测评
    - 烤鸭：🌶️ "能吃，但不是那种爽"
    - 涮羊肉：🌶️🌶️ "蘸麻酱这事我到现在还没习惯"
    - 豆汁：🌶️🌶️🌶️🌶️🌶️ "别试"
    - 等等——用你的真实感受

视觉处理：
  地图区域用暗色版高德地图
  测评用横向滚动的卡片，每张卡一个大 emoji + 辣椒评分 + 一句话
  红色强调在"好吃的湘菜"相关元素上
```

### Section 6 — "周末去哪"（探索）

```
大标题："北京不只有故宫和长城"

你的个人推荐，不是旅游网站的百科：

  - 颐和园：哪个季节去最好、哪个角落没人
  - 胡同：你最喜欢逛的几条
  - 798：作为一个社会学学生怎么看
  - 小众地：只有你知道的好地方
  - 周边：周末两日游推荐

视觉处理：
  照片墙风格（masonry grid）
  每张照片做了暗角处理，统一暗色调
  hover 时照片变亮 + 地点名称浮现
  点击进入详情
```

### Section 7 — "北京人的规矩"（文化）

```
大标题："在北京，有些事湖南人真的不懂"

你作为社会学学生的观察（这个身份特别好，有说服力）：

  - 北京人说话方式 vs 湖南人
  - "您"这个字的文化重量
  - 饭局上的规矩差异
  - 湖南口音在北京
  - 在北京交朋友/谈恋爱

视觉处理：
  这一节用更"学术笔记"的排版
  左边大字是"现象"，右边小字是"社会学观察"
  像田野笔记一样的版式
  偶尔插入你的真实对话/故事
```

### Section 8 — 页脚 / 关于

```
简洁：
  大标题："关于这个网站"
  一段话介绍你自己 + 为什么做这个网站
  一个联系入口（邮箱、社交媒体）
  底部一行小字：Made with 🌶️ by a Hunan person in Beijing

视觉：
  背景最深（#0A0A0A）
  文字居中、大留白
  可能有微弱的辣椒图案作为背景装饰
```

## 四、动画系统

```
1. 滚动触发（核心动画）
   - Framer Motion useScroll + useTransform
   - 内容从下方淡入 + 上浮 40px
   - 图片/卡片从侧边滑入
   - 每章触发的时机：元素进入视口 30% 时

2. 微交互
   - 链接 hover: 颜色过渡到辣椒红，300ms ease
   - 卡片 hover: translateY(-4px) + 红色边框出现
   - 按钮 hover: 红色背景从左边滑入填充

3. 视差
   - 部分背景元素以 0.5x 速度跟随滚动
   - 让空间感更强

4. 页面过渡（详情页之间）
   - 简单的淡入淡出
   - 或者一个红色" swipe "过渡
```
